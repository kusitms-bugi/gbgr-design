# AGENTS.md - packages/react

## Scope
- This file applies to everything under `packages/react`.

## Goal
- Maintain production React components built on `@gbgr/react-headless` and shared icons.

## Preferred Workflow
1. Keep component API and exported paths stable.
2. Run `pnpm --filter @gbgr/react build` after component or export changes.
3. Run `pnpm --filter @gbgr/react typecheck` before finalizing.
4. Confirm CSS export files referenced in `package.json` still exist.

## Guardrails
- Do not bypass headless primitives in ways that duplicate interaction logic unnecessarily.
- Avoid breaking changes to public component props without clear migration intent.
- Keep `dist/` generated by build; do not hand-edit.

## Focus State Convention
All interactive components must follow these focus ring rules:

### Pattern A: Standalone elements (Button, ToggleSwitch, SegmentedControl)
For elements that are their own interactive area:

```css
/* 1. 기본 outline 제거 */
.gbgr-component {
	outline: none;
}

/* 2. ::after로 커스텀 포커스 링 */
.gbgr-component::after {
	content: "";
	position: absolute;
	inset: -4px;
	border: 2px solid transparent;
	border-radius: var(--radius-full);
	pointer-events: none;
	transition: border-color 120ms ease;
}

/* 3. focus-visible에서만 브랜드 컬러 표시 */
.gbgr-component:focus-visible::after {
	border-color: var(--color-semantic-accent-brand-default);
}
```

### Pattern B: Embedded controls (Checkbox, Radio)
For elements with a small control area inside a wrapper (20×20 control, 16×16 inner):

```css
/* 1. Wrapper에 outline 제거 */
.gbgr-component {
	outline: none;
}

/* 2. Control의 ::after로 포커스 링 */
.gbgr-component--style:focus-visible .gbgr-component__control::after {
	content: "";
	position: absolute;
	inset: -2px;
	border: 2px solid var(--color-semantic-accent-brand-default);
	border-radius: 8px; /* checkbox */ or 9999px; /* radio */
	pointer-events: none;
}
```

### Key rules
- `outline: none` on base element to remove browser default (파란색 기본 포커스 제거)
- `border: 2px solid` with brand color (`#ffc61a`) on `focus-visible`
- `pointer-events: none` on the ring so it doesn't interfere with clicks
- Only `focus-visible` (not `focus`) so mouse clicks don't show the ring
- `::before` for hover state layer, `::after` for focus ring — they coexist
- Pattern A: `inset: -4px` for 4px gap from element edge
- Pattern B: `inset: -2px` on control for 2px gap (smaller touch target)

## Deploy
1. Component API or export changes require a changeset: `pnpm changeset`.
2. Build and typecheck must pass before release (`pnpm --filter @gbgr/react build` and `typecheck`).
3. Coordinate breaking API changes with docs updates in the same release cycle.

## Testing
1. Run `pnpm --filter @gbgr/react typecheck` and `pnpm --filter @gbgr/react build` for every component change.
2. If component behavior changes, add or update tests when test infrastructure is available.
3. If no automated tests are available for the changed component, record a manual interaction checklist in PR.
4. Write `describe` and `it` test descriptions in Korean.

## Quick Commands
- Build: `pnpm --filter @gbgr/react build`
- Typecheck: `pnpm --filter @gbgr/react typecheck`
